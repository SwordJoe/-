1. SIMCV_imwrite
    写入yuv420p格式，图片变蓝
    =====已解决，原因是原来的代码进行格式转换的时候，调用的宏定义的COLOR_YUV420p2BGR，但这个宏实际上转换的是YV12的格式，所以会显示蓝色======
    =====正确的宏应该是COLOR_YUV2BGR_I420======

2、 SIMCV_convertPixelFormat
    rgb24、bgr24、nv21没问题
    转换为nv12后，写入jpg图片，图像变蓝，用ffmpeg生成的nv12写入jpg没问题        ====已解决，原因是IMG_RGB2YUV420SP()这个函数开了宏定义判断分支，但是永远走的是同一个分支，重新用if()else()=======
    转换为yuv420p后，写入jpg图片，图像变蓝，用ffmpeg生成的yuv420p写入jpg同样变蓝  ====已解决,原因同上====

3、SIMCV_convertPixelFormatWithScale
    ####该函数只支持从NV12、NV21、YUV420P、YUYV422、BGR24五种颜色空间转为RGB24，没有其他的转换####
    问题1：yuv420p格式，生成的测试图片颜色不太正常
    问题2：rgb24格式，偶尔会不正常，比如从(50,50)裁剪宽200高200的区域，x_scale=y_scale=2时，生成图片乱码
    问题3：NV12、NV21变蓝，原因还是因为开启了宏定义的判断分支，测试时永远走的是NV21的分支，所以测试NV21的时候是正常的，测试NV12的时候图片就会变蓝

4、SIMCV_resize, nv12、nv21、yuv420p转换后图片显示绿色
    ======已解决，nv12、nv21、不能直接用opencv的resize()函数转换=======
    ======解决办法是将nv12、nv21的Y、U、V三个分量提取出来，分别对这三个分量进行resize()，然后再将resize()后的Y、U、V分量合并======


5、SIMCV_vconcat、SIMCV_hconcat，拼接YUV格式的时候，颜色不对
    =====已解决，原因是源代码用opencv的vconcat和hconcat直接进行拼接，对rgb有用，但是对yuv格式的没有用=====
    =====同SIMCV_resize一样，需要将YUV分量提取出来，单独拼接，最后再合并成一份YUV数据=====

6、SIMCV_LibyuvConvert::convert
    yuv420p裁剪，转成任何格式都不对
    nv12裁剪，转成yuv420p颜色不对，rgb24和bgr24不正常
    nv21裁剪，转成yuv420p正常，转成rgb24和bgr24不正常
    =====这个函数包含了 裁剪 + 缩放 + 颜色空间格式转换 三个功能，要做的事情有点多，之前的实现会导致生成的图片颜色有点不正常=====
    =====可行的方法是，先将各种YUV格式转换成I420, 再利用libyuv::ConvertToI420()进行roi区域裁剪，裁剪完了再利用I420Scale()缩放，最后转换颜色空间格式=====

7、SIMCV_drawColorLine, 在YUV格式上画线时R、G、B乱序，指定 255 0 0 时，画出的线是绿色，指定0 255 0 时，画出的线是红色
   ####该函数只支持NV12、NV21、YUYV422、RGB24、BGR24五种像素格式####
   原因是在几种YUV上画线的时候，传入的参数是R、G、B，但是赋值的时候没有进行R、G、B到Y、U、V的转换，直接就将RGB的值赋给YUV了

8、SIMCV_LibyuvConvert::convertPixelFormat
    yuv420p --> nv12 ok
            --> nv21 蓝色问题               =====已解决，原因是原代码NV12和NV21用的同一个libyuv接口I420ToNV12()，NV21应该用I420ToNV21()=====
            --> rgb24 ok
            --> bgr24 ok
    
    nv12    --> rgb24 ok
            --> bgr24 ok
            --> yuv420p ok            
    
    nv21    --> rgb24
            --> bgr24
            --> yuv420p 蓝色问题        =====已解决，原因同上，NV12和NV21用的是同一个接口=====
